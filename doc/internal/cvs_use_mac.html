Using Cvs on Macintosh

Introduction

The cvs client is supported on both powermac and 86k mac. This page is a guide on how to download and install cvs on your mac. It is very important that you read the "known problems" section, before using maccvs.

The cvs client is use from within MPW. The synopsis is the usual, for example:

cvs get guienv

Requirements

  1. You must be a member of the netgroup `beta' You can check whether you are in the group by executing the following in the shell: 
     ypmatch beta group
    If you are not in the group, contact the staff.

  2. You must have an "~/.rhosts" file in your home-directory on unix. It should contain a line with the name of the mac, and your unix username. the file must have permissions "600" = "-rw-------", so that only the user can read it. Create it with a unix texteditor, as it should contain unix linefeeds. An example: 
    saljut beta
    The above line is added to the /users/beta/.rhosts file. If you use the beta username, this is where you should add your own line.

  3. The owner name of your mac must be the above username. Change this in the "Sharing Setup" control panel.

Software

  • MPW: You can get this from the ETO disc.

  • MacPerl: Perl for macintosh. This is included in the package you are going to download.

  • MacCvs application and scripts, also included in the package.
Download the package: Maccvs Installation

Installation

  1. Doubleclick on the downloaded self extracting archive to extract the files.

  2. Doubleclick on the file "Mac_Perl_513r2_tool" to extract the MacPerl folder required for your macintosh.

  3. Drag userstartup·perl and userstartup·cvs into your MPW folder.

  4. Start MPW, and select the MacPerl folder and the MacCvs folder when asked to do so. There is two MacCvs folders, one for power mac and one for 68k macs. Be sure to choose the correct one for your machine.

Known problems

  • The file names cannot be longer than 26 characters.

  • The "cvs commit" command requires the "-m" option and one word as argument - not even quoted strings will do - (e.g. cvs com -m Fixed_Mac_error, but not cvs com -m "Fixed Mac error"), otherwise the application will crash.

  • There is currently an error in the userstartup·cvs script: It adds an extra space after the last comma in the setup of the commands variable. This will prevent the MPW shell from finding programs in the current directory. Fix the error in the startup file and restart MPW. Of course the package should be updated...

  • The Maccvs runs as a background application. If you interrupt a cvs command in MPW, you may have to restart the computer to get CVS to work again, depending on what the background application was doing at the time of the interrupt.
