Automagic creation of distributable system using InstallShield

Do the following:
  1. Generate packingscripts on UNIX 
    cd $BETALIB\export\dotar
.\windows.dotar -pe
    This generates a bunch of scripts in $BETALIB\export\tarfiles\nti.pe

  2. Copy the pack-scripts from UNIX: 
    (UNIX) $BETALIB\export\tarfiles\nti.pe
    to 
    (TUBORG) %betalib%\export\nti_%MIASDK%
  3. Make sure you have the minimal install shield stuff installed . There is a zip file in ~beta on noatun. Place it in d:\installshield. If you have to put it somewhere else you will have to edit scripts to make it work.
  4. Generate images

    This is where the procedure for old and new Installshield differ. Old Procedure 

    (TUBORG) cd %betalib%\export\nti_%MIASDK%/
(TUBORG) perl %betalib%\bin\admin\autogenerate-installshield %MIASDK% *.cmd
    The result will be, that the proper disk images are located in: 
      (TUBORG) %betalib%/export/nti_%MIASDK%/datafiles/disk*
    and that these also are packed into one ZIP file, located in: 
      (TUBORG) %betalib%/export/nti_%MIASDK%/datafiles/beta_%MIASDK%.zip
    Finally, the file 
      (TUBORG) %betalib%/export/nti_%MIASDK%/datafiles/beta_%MIASDK%.lst
    will contain the output from the packing process (check it to find possible packing errors, using "grep -i error").

    New Procedure

    Run 

    cd %betalib%\export\nti_%MIASDK%/
perl extractexportfiles %MIASDK% *.cmd

  5. Copy the ZIP file to UNIX

    6. Place it in the directory: 

       (UNIX)  $BETALIB/export/tarfiles/FTP/
