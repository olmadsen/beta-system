Getting a WIN32 development enviroment for developing BETA.

To use your Windows system for BETA development you need the following software installed.
  • SSH
  • CVS
  • Perl
  • gcc
  • unzip (optional)
  • gdb (optional)
  • Visual Studio (for an MS-based system only)
  • Misc tools (automatic)

You don't need any special shells or Cygwin tools.

When you have finished installing software, you can get your development track going with the instructions here.

SSH

We recommend openSSH or Putty, which both have commandline based ssh program (called ssh.exe and plink.exe respectively). See more on this Danish-language SSH setup page.

CVS

We now recommend WinCVS, which has a commandline based program called 'cvs'. See more on this Danish-language CVS setup page.

Previously, we have used the version of cvs called cvss.9-10-ssh.exe. This was installed by renaming it to be called cvs.exe and placing it somewhere in your path. It's not Y2k compliant and we can't find the source, but it does understand the CVS_RSH variable, which is rare on Windows.

Perl

You can find perl in
http://www.mjolner.com/Quality/software/software/Pc/Misc/Perl/ActivePerl-5.6.0.623-MSWin32-x86-multi-thread.msi
or you can get it directly from ActiveState. We normally use version 5.6, but there are no known problems with later versions.

There is also a GUI frontend available:

http://www.mjolner.com/Quality/software/software/Pc/Misc/Perl/Win32-GUI-0.0.558-PPM-5.6.zip

GCC

You need GCC and associated tools. If you are doing GNU-based development then you will be using the C compiler, linker and assembler. If you are doing MS Visual Studio based development you still need the GCC assembler (gas) for building betarun, the runtime system. (This only applies to developers. End users of the MS version of the BETA system don't need GCC installed).

We use the MingW32 version of gcc, which is a native port to Windows, not the Cygwin version. MingW32 is available in two versions, CRTDLL, and MSVCRT, which use two different C runtime libraries. We always use the CRTDLL version.

You can get a self-installing exe file from a random release CD of the BETA system, or from our own download area at

ftp://ftp.mjolner.dk/mjolner-system/latest/utils/windows/GNU/gcc-2.95.2-qf-runtime-2000-03-27.exe
If you decide to download directly from the MingW32 project, be aware that we have tested with version 2.95.2 with the new runtime system from 2000-03-27. Newer versions may or may not work.

The only necessary environment variable you need to set is PATH, which typically needs to be set to:

PATH=C:\gcc-2.95.2\bin;%PATH%

(replace the '%PATH%' with whatever was in PATH before unless you are setting it in a .bat file).

You can set this environment variable in Start->Settings->Control Panel->System->Environment (in NT) or in c:\autoexec.bat (on Windows 98). Beware of trailing spaces!

Unzip (optional)

We normally use Powerarchiver.

GDB (optional)

This is the GNU debugger, and it can be useful for debugging, especially if you have a GNU-based BETA development system. You can compile programs with the flag "-s 88" to make executables that can be debugged with gdb.

You can find gdb-4.17 at http://www.mjolner.com/Quality/software/software/Pc/Utils/GNU/ or a somewhat newer version at ftp://ftp.franken.de/pub/win32/develop/gnuwin32/mingw32/porters/Mikey/.

MS Visual Studio (optional)

If you want to develop the MS version of the BETA system, then you need to install MS Visual Studio. You must ensure that the C compiler, cl, is in your PATH. The installer does not do that.

Miscellaneous tools

There are miscellaneous GNU tools already committed into bin_ntiwrappers and therefore automatically available to BETA developers. The source files are in the subdirectory gnusrc in order to comply with the GPL. You can delete this directory to save space, but it will return every time you do 'cvs update -d' (so don't do that).

These utilities are compiled by the Mingw32 project, therefore they are 'native' Windows versions, and do not need any special DLLs. Most of them are from ftp://ftp.franken.de/pub/win32/develop/gnuwin32/mingw32/porters/Mikey/.

They include GNU make, patch and diff, cut, find and xargs, fmt, head and tail, md5sum, sum, tr, wc, sort and uniq. There is also a minimal shell for use with GNU make, shmsdos, courtesy of the Mozilla project.

They all understand the --usage option, but no documentation is installed by default. Documentation is probably available on a random Linux machine, through Google or by looking in the source files.

A word on shells and Cygwin

We have seen various difficult to understand/reproduce errors caused by people who install Cygwin in order to have a more comfortable shell. If you install Cygwin, you must be sure that it is later in your PATH than the tools described on this page. The problems include, but are not limited to Cygwin software that expects newlines at the end of each line (instead of the Windows cr-lf convention) and executables that won't work on non-Cygwin machines.

If you want the command line (DOS box) on Windows to be more comfortable or Unix-like we can recommend the program 'tweakui', available through Google and here which lets you activate a lot of usability features.
