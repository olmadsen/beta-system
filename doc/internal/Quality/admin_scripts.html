bin_admin script documentation

This document tries to explain some common uses of the admin scripts for the Mjolner system.

StripObsolete?
WhichFinds matching commands in path
autogenerate-installshieldCreates install program on nti. Ordinary use
checkbetCheck recursively that every .ast file has a corresponding .bet file. (bigendian Unix only)
copydisksObsolete?
cvschecklocalCheck recursively (starting from current directory) if 'cvs update' would change anything (same as cvs -nq update -Pd)
cvscomObsolete? (Sun only) does 'cvs com' after doing a lot of diff'ing(?)
cvsupdatelocalGet latest version from CVS starting from current directory (same as cvs update -Pd)
diff-versionsrun 'diff' on files in two directory-trees
dirlistInterface to dirlist.txt. ORIGIN of many mbs scripts.
dirlist.txtText file containing a table listing all directories in the MBS
env.perlSets up a lot of variables. Should closely match ~beta/configuration/env.csh
findexeRecursively list files with execute permissions (unix only)
hack-prettyObsolete?
install_student_betaHelps install MBS in usr/local/lib. Must be edited before use.
mailto(unix only) 'mailto recipient subject' then type letter, end with Ctrl-D.
mbs_allfilesList all files in MBS into ~beta/AllFilesInMBS.lst
mbs_checkbetCheck that all .ast[L] files in MBS has a .bet file
mbs_checkdbCheck that all .bet files has a .db file
mbs_checkpackLists all files in MBS that have not been packed (Lists far too many, but is a first step towards a strong check)
mbs_cleanRemoves garbage files from MBS (*~ *.dump, etc.)
mbs_compileRuns the beta compiler on all .bet files in all dirs that have the compileflag set in dirlist.txt
mbs_compile_dirsLists all directories where compileflag is set (for what use?)
mbs_compiletoolsBuilds and installs all the executables needed for distribution
mbs_cvscheckupdateChecks whether there are any files that need updating. The same as cvs -nq update -Pd in $BETALIB
mbs_cvscommitCheck if there are local changes in MBS and prompts before doing 'cvs commit'.
mbs_cvsdiffRuns 'cvs diff' throughout MBS. The same as 'cvs diff -u'.
mbs_cvstagSet/remove/move/ tags, including branchtags on MBS. You can acheive the same effect with cvs commands from top dir.
mbs_cvsupdateUpdates track to newest files from CVS. The same as cvs -q update -Pd in $BETALIB
mbs_freezeMakes all files in MBS read-only. On unix, also makes directories read-only
mbs_grepRuns a case-insensitive search through all source-files in MBS (
mbs_etagsCreates an Emacs TAGS file for all source-files in MBS (
mbs_grepastgrep in all ast files
mbs_list(unix only) run 'ls -F -R' in all base directories in MBS
mbs_rmastRemove all /ast[l] files in MBS, excluding grammars
mbs_rmcodeRemove all codefiles (object, assembler and debug) from the machinedirectories given as arguments
mbs_setallversions(Obsolete) List versions for all modules in MBS
mbs_unfreezeRemove read-only from all files and dirs
mbs_mkbootPutting new files in the boot directory
mbs_commitCommitting in Betarun and the compiler
mbs_makeBuilding Betarun
mbs_mkbootPutting new files in the boot directory

setup.rul.templateTemplate used when generating the installer for nti
stat(obsolete?) toolstat wrapper?

utils.perlSubroutines used by env.perl and dirlist
