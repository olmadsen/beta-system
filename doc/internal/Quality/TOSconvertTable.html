Low Level Operations Conversion Table

The following table gives an overview of conversions from the old TOS primitives to the new % primitives.

[ bit-ops ] [ byte-ops ] [ short-ops ] [ long-ops ]

Bits:
Old TOS command Equivalent new %-command Notes
Get:
X -> TOS'%getBits[bitNo,noOfBits]' (bitNo,noOfBits) -> X.%getBits  
X -> TOS'%getSignedBits[bitNo,noOfBits]' (bitNo,noOfBits) -> X.%getSignedBits  
Put:
(@@X,val) -> TOS'%putBits[bitNo,noOfBits]' (val,bitNo,noOfBits) -> X.%putBits  
Shift:
X -> TOS'%shiftRight[noOfBits]' X %srl noOfBits  
X -> TOS'%shiftLeft[noOfBits]' X %sll noOfBits  
N/A X %sra noOfBits  
N/A X %srl noOfBits  
N/A X %ror noOfBits  
N/A X %rol noOfBits  
Logical:
X -> TOS'%not' %Bnot X  
(X1,X2) -> TOS'%and' X1 %Band X2  
(X1,X2) -> TOS'%or' X1 %Bor X2  
(X1,X2) -> TOS'%xor' X1 %Bxor X2

Old TOS command Equivalent new %-command Notes
Get:
X -> TOS'%getByte[byteNo]' byteNo -> X.%getByte  
X -> TOS'%getByte' 0 -> X.%getByte  
X -> TOS'%getSignedByte[byteNo]' byteNo -> X.%getSignedByte  
X -> TOS'%getSignedByte' 0 -> X.%getSignedByte  
(R,byteNo) -> TOS'%inxGetByte' byteNo -> R.%getByte (1)
(R,byteNo) -> TOS'%inxGetSignedByte' byteNo -> R.%getSignedByte (1)

addr -> TOS'%adrGetByte[ByteNo]' %getByteAt(addr+4*ByteNo) (12)

@@X -> TOS'%adrGetByte' 0 -> X.%getByte (10)

Put:
(R,byteNo,val) -> TOS'%inxPutByte' (val,byteNo) -> R.%putByte (1) (2)

(addr,X) -> TOS'%putByte' X %putByteAt (addr) (11)

Old TOS command Equivalent new %-command Notes
Get:
X -> TOS'%getShort[shortNo]' shortNo -> X.%getShort (4)

X -> TOS'%getSignedShort[shortNo]' shortNo -> X.%getSignedShort (4)
X -> TOS'%getSignedShort' 0 -> X.%getSignedShort  
(R,byteNo) -> TOS'%inxGetShort' (byteNo div 2) -> R.%getShort (1) (3)

addr -> TOS'%adrGetShort[ShortNo]' %getShortAt(addr+2*ShortNo) (4) (12)
@@X -> TOS'%adrGetShort[ShortNo]' ShortNo -> X.%getShort (10)
addr -> TOS'%adrGetShort' %getShortAt (addr) (12)
@@X -> TOS'%adrGetShort' 0 -> X.%getShort (10)

@@X -> TOS'%adrGetSignedShort ' 0 -> X.%getSignedShort (10)

(R,byteNo,val) -> TOS'%inxPutShort' (val,byteNo div 2) -> R.%putShort (1) (2) (3)

(addr,val) -> TOS'%putShort[shortNo]' (val) -> %putShortAt (addr+2*shortNo) (11)
(@@X,val) -> TOS'%putShort[shortNo]' (val,shortNo) -> X.%putShort (4)(10)
(addr,X) -> %TOS'%putShort' X %putShortAt (addr) (11)

Long:
Old TOS command Equivalent new %-command Notes
Get:
X -> TOS'%getLong[longNo]' longNo -> X.%getLong (6)
X -> TOS'%getLong' 0 -> X.%getLong  
(R,byteNo) -> TOS'%inxGetLong' (byteNo div 4) -> R.%getLong (1) (5)
Adr Get:
addr -> TOS'%adrGetLong[longNo]' %getLongAt (addr+4*longNo) (6) (12)
@@X -> TOS'%adrGetLong[longNo]' longNo -> X.%getLong (10)
addr -> TOS'%adrGetLong' %getLongAt (addr) (12)
@@X -> TOS'%adrGetLong' 0 -> X.%getLong (10)
Put:
(R,byteNo,val) -> TOS'%inxPutLong' (val,byteNo div 4) -> R.%putLong (1) (2) (5)
Adr Put:
(addr,X) -> %TOS'%putLong' X %putLongAt (addr) (11)
(@@X,Y) -> %TOS'%putLong' (Y,0) -> X.%putLong (10)
(addr,X) -> %TOS'%putLong[longNo]' X %putLongAt (addr+4*longNo) (6) (11)
(@@X,val) -> TOS'%putLong[longNo]' (val,longNo) -> X.%putLong (6) (7)(10)

Notes

  1. R is a repetition.
  2. Note the swap of parameters.
  3. Note, that short selection in the new format is using short numbering, where the old format used byte numbering.
  4. Here ShortNo is really a short-index in both old and new syntax (unlike (3)).
  5. Note, that long selection in the new format is using long numbering, where the old format used byte numbering.
  6. Here longNo is really a long-index in both old and new syntax (unlike (5)).
  7. Notice that (val,0) -> A.%putLong is the same as val -> A.
  8. Notice that (val,@@X) -> %putValAt (insecure) is the same as (val,0) -> X.%putVal (secure), but not for X dynamic reference or array
  9. Notice that (@@X) -> %getValAt (insecure) is the same as 0 -> X.%getVal (secure), but not for X dynamic reference or array
  10. Secure special case for the preceding insecure rule
  11. Notice that
    val %putByteAt  (@@R[inx]+off) is the same as (val, (inx-1)*E+off)->R.%putByte (avoid use of @@), and
    val %putShortAt (@@R[inx]+off) is the same as (val, ((inx-1)*E+off) div 2)->R.%putShort for E>=2 (avoid use of @@), and
    val %putLongAt  (@@R[inx]+off) is the same as (val, ((inx-1)*E+off) div 4)->R.%putLong for E>=4 (avoid use of @@)
    where E is the element size (in bytes) of the repetition R
  12. Notice that
    %get(Signed)ByteAt  (@@R[inx]+off) is the same as ((inx-1)*E+off)->R.%get(Signed)Byte (avoid use of @@), and
    %get(Signed)ShortAt (@@R[inx]+off) is the same as (((inx-1)*E+off) div 2)->R.%get(Signed)Short for E>=2 (avoid use of @@), and
    %getLongAt          (@@R[inx]+off) is the same as (((inx-1)*E+off) div 4)->R.%getLong for E>=4 (avoid use of @@)
    where E is the element size (in bytes) of the repetition R
