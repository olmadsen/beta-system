Creating Hybrid CD with the Mjølner System

The procedure described below has been replaced with the procedure described in unixfreeze.html

The following is a description of the steps required to produce a Hybrid CD (i.e. a CD that can be used on both UNIX, Macintosh and PC), with the Mjølner System.

The actual burning of the CD takes place on Ran, using the HyCD program, but several things must be done on other computers - see below.

Text files

Text files (with extension .txt) should be stored with PC linebreaks (CR-LF). This will be readable on all three platforms. However, if a text file is only relevant to macintosh or UNIX, it may be stored with linebreaks for those platforms.

Packing Macintosh files

A number of special arrangements must be done to successfully pack macintosh files from the PC.

Specify filename-extension-to-file-type/creator mapping

For each type of file, that will be visible on the CD once opened on a macintosh, you should secify a mapping between the filename extension and the macintosh File Creator and Type. On a macintosh, you can fetch these informations using ResEdit: Open the file, you want to get information about from the Get File/Folder Info menu item on the File menu.

In HyCD, you can then specify these mappings in ...

Macintosh files with Resource Fork

To get a macintosh file with resource fork stored correctly on the CD, you must do as follows. Most macintosh files have a resource fork (physically in a separate disk entry, but on the mac never explicitly visible in the Finder), e.g. all programs, but for instance BinHed'ed files (ending in .hqx have no resource fork by default.
  1. Create the files you want stored on the macintosh.
    If possible, use DOS 8x3 file names. Then the file names will appear correctly on PC too. If you do not use 8x3 names, the filenames will appear garbled on PC, but will still appear correctly on Macintosh. Specificly the toplevel directory name of the macintosh stuff you want to include on the CD should have a name of at most 8 chars (e.g. macintosh), since this will normally be visible on PC/Unix next to the directories for the other platforms This 8x3 limitation is not caused by the PC software, but by the Macintosh interface to PC disks (PC Exchange) only supporting DOS file systems, and not Windows NT file systems (should be fixed in MacOS 8.1 - not tested).
  2. Mount a DOS formatted PC disk (or Syquest) on the Macintosh.
  3. Copy the macintosh files to this disk from the Finder.
  4. Mount the disk on PC
  5. Copy the toplevel macintosh file to the PC.
    Copy everything in the macintosh directory to the directory, where you are preparing the CD image. There will normally be a directory named Resource.frk and a file named Finder.dat in each macintosh directory, but other special files may be present too. Observe also that files/directories which does not have 8x3 names will have garbled file names on the PC.
  6. If you copy a deep structure from the mac, where the macintosh specific stuff does not appear at the top of this structure, you only need to copy the special files from the macintosh toplevel directory and downwards. Assuming, that the generic parts have 8x3 names.
