When a new release is created/installed do the following: 

============ on daimi.au.dk ==============

.  Update the BETASTAT and BETAREPORT variables in the tool scripts:
   /usr/local/lib/beta/rx.y/bin
   beta and mjolner. Default is 'no'
like this

BETASTAT=${BETASTAT-yes}
BETAREPORT=${BETAREPORT-yes}
BETAREPORT_EMAIL=${BETAREPORT_EMAIL-yes}
BETAREPORT_WWW=${BETAREPORT_WWW-no}

. Update the RELEASE variable in exectool

. Update the INSTALLATION variable in exectool to 'student' 
  INSTALLATION=student   # Must be either student or beta.


=========== on mjolner.dk ============

. Update /users/ess/.procmail_betastat (ask Morten :-))

. Create a directory for the release e.g.
  cd /users/ess/beta/Statistics
  mkdir beta-r5.2.2
  mkdir student-r5.2.2
  mkdir beta-beta-report-r5.2.2
  mkdir beta-mjolner-report-r5.2.2
  mkdir beta-report-r5.2.2
  mkdir student-beta-report-r5.2.2
  mkdir student-mjolner-report-r5.2.2

. cd student-r5.2.1/mjolner
  touch mjolnerstat
  touch index.html
  ln -s ../../student-mjolner-report-r5.2.1 reports

. cd student-r5.2.1/beta
  touch betastat
  touch index.html
  ln -s ../../student-beta-report-r5.2.1 reports

Now the logs are ready. This is important for getting proper statistics from the start.

-----------------------------------------

To prepare the www presentation of the statistics (can wait until you have time, 
since the data are now collected in the logs):

. Update the symbolic links in /users/ess/public_html/Statistics/
   e.g.:

    	student -> /users/ess/beta/Statistics/student-r5.2.1

. cd student

  The following files should be fetched from the statistics repository, 
  but since this is not properly installed, copy from last semester:
  
   cp ../student-r5.2/frontpage.html .
   cp ../student-r5.2/frontpagestart.html .
   cp ../student-r5.2/index.html .
   cp ../student-r5.2/tools.html .

  Update the release information in these files
  

. To test the www script:

   cd /users/ess/toolstat
   statwwwall
  
   and see the result in the www pages: 

   http://www.mjolner.dk/~ess/Statistics/student/index.html or 
   http://www.mjolner.dk/~ess/Statistics/beta/index.html

. ~ess/.crontab runs the statistics on ull

. Regarding statistics archive a number of things must be done
  at each semester start (these are currently not automated):
   1. Update 'semester' in in statweekall
   2. Add the new semester directory in 
      statlocation=$statarchive/$tool/$semester
   3. Add a 'Statistics archive' entry in tools.html (if not present)
   4. Add semester entries in 
      http://www.mjolner.dk/~ess/Statistics/archive/index.html
   5. In 
      http://www.daimi.au.dk/~beta/Statistics/archive/student/<tool>/<semester>
      Copy and update: index.html, weeks_start.html and weeks_end.html
      from the previous semester.
