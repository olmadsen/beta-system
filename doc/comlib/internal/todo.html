Less Important stuff

  1. Support for all the CoXXXX functions.
  2. Text interface
    1. Compileren kan i øjeblikket ikke håndtere ^text som formel parameter til en COM virtuel. Der genereres derfor [0]@char istedet.
    2. Elvis og hardcore siger der er et Istring interface Dette bør måske være den form for tekster vi bruger i komponenter vi selv implementerer.
    3. Der er lavet et BSTR pattern, integreret med pattern wtextHolder. Et problem eksisterer dog for [in] parametre af typen BSTR. Disse er nødt til at være erklæret som int32 istedet for BSTR, da compilerne ikke accepterer BSTR som formel enter parameter.
      Se kommentar i holders.bet for mulig anden implementation.
  3. Call by value of data 
    foo:< (# r: @myData enter r ....
    Now allowed in compiler, but not tested.
  4. Pointer-to-pointer types in general (can wait)
  5. Special types (not needed to get started, though)
    1. sequence
    2. any (not a MIDL type, though...)
    3. float
    4. hyper
    5. ...
  6. Enums cannot be IntegerObjects, since IntegerObjects cannot be parameters to external functions.
  7. The current definition of HRESULT cannot be used for external parameter in BETA. Thus the representation used for the HRESULT parameters are currently "int32", and it is up to the user to convert it to something meaningful using the HRESULT pattern.
  8. The VARIANT data type will not work as an in-out or out parameter, since the data is inlied in the holder. The solution should probably be something like the solution for GUID.
    Morten is working on a new design...
