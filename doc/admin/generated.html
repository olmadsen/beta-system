MIADOC: What is/can be auto-generated

The aim of the MIADOC "project" is to generate as much of the documentation as possible automatically, and only having to update the raw HTML pages with the "substance" of the manuals.

Currently this aim is partly obtained. Here is a list of what is currently auto-generated.
The next section caontains a list of what is notyet generated, but is planned to be

Generated files

The following is currently generated when issuing a make command on a UNIX command line from the doc directory. Most of the stuff may also be generated from a PC (it's in perl).
  1. interface descriptions
    These are generated from the BETA interface files in BETALIB. See doc/how_to.html#makehtml_idx for an explanation.
  2. Man pages
    The manpages in HTML format in doc/manpages are generated automatically from the NROFF files in BETALIB/man/man1.
  3. Grammars
    The grammars in HTML format in doc/grammars are generated automatically from the grammars in BETALIB/grammars.
  4. list of programs
    A list of programs can be generated using doc/bin/make_program_list.perl. Currently used by e.g. Xwindows and Basiclib. See the Makefiles the respective places for example usage. Also see doc/templates/programs.html
  5. list of figures
    A list of figures can be generated using doc/bin/make_figure_list.perl. Currently used by e.g. Bifrost See the Makefile for example usage. Also see doc/templates/figures.html
  6. Marked-up beta-code
    The program doc/bin/beta2html.perl can be used to generate an HTML file with a marked-up version of a beta file, e.g., a demo program for an appendix. Used in e.g., Bifrost Tutorial. See the Makefile for example usage.
  7. BETA generated pages
    BETA programs can also be used to generate the documentation. An example is the generation of the Appendix for the BETA compiler reference manual, which describes the possible semantic errors and warnings the compiler can cause. The program used for generating this is in the file doc/compiler/files/MakeMsgList.bet. The stuff for producing the header and footer should be fragmented out, if other BETA programs are used in the future.
  8. Consistency check for dead links
    The program doc/bin/linklint_2.1/linklint validates that all referenced URL's used in this documentation are valid. It does not verify the HTML itself (there are several places on the web this can be done). The result can be seen in doc/admin/errors/linklint.html. The style sheet can be validated as specified in doc/admin/index.html#css-validate.

Files not yet auto-generated (but planned to be)

In future releases the following will be possible to auto-generate:
  1. Table of contents
    This is intended to generate in a separate frame.
  2. Index
    This is intended to be generated by searching for index anchors in the substance files. Should be merged with the index generated for interface descriptions (like in the word versions where index entries from interface descriptions appeared in italic, and entries from the body files appeared in roman).
  3. Navigation buttons
    The Next, Prev, etc. buttons are tedious to set up and maintain by hand. As described in doc/admin/todo.html a proposal exists for genrating these automatically in a separate frame.
