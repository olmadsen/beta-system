    4. Maybe some of the first converted manuals has gives rise to too many HTML files. In the last converted manuals, only Heading 1 and Heading 2 yielded new files, but in the first converted manuals, all heading levels yielded new files. Merge HTML files by moving the contents to another file, then fixing all references to the now emptied file (use, e.g. tags-query-replace in Emacs.
    5. Check images. You may want to add ALIGN={left,middle,right} to the images. Also you may want to make background transparent. Most often this can be done by the UNIX command giftrans -T foo.gif > xxx.gif; mv xxx.gif foo.gif. Also some of the images converted by WebMaker sometimes look a little "blurred". Fix this by copying the original images from the original Word files (still present in /users/beta/doc/miadoc/REPOSITORY) into a graphics program, that can save as GIF (e.g. GraphicConverted on Macintosh). You may also want to add WIDTH and HEIGHT attributes to image tags to fasten up the browser's layout process.
    6. Fix references/bibliography. As specified in doc/admin/rules.html references to "our own stuff" should be via the shared doc/bibliography/index.html file. Most of the original Word files contained a list of references to [Madsen 93], [Knudsen 94], [MIA XX-XX]. These been fixed in the manuals. Please check remaining references and bibliographies.
    7. The WebMaker generated files often contain a "local TOC" for subsections (either in same file as section, or in separate files). I think these may confuse people, and should probably be removed? They are marked with <!-- TOC --> in the HTML.
    8. You may also want to rename some of the files, which are named, e.g. basiclib-3.html, basiclib-4.html etc. to more descriptive names. After the rename, issue an etags *.html in your directory and then inside Emacs issue a tags-query-replace replacing the old file name with the new one. Do this for each file renamed. I did this for, e.g., doc/bifrost-tut.
