    In the process of doing this, I (Peter) have learned various tricks during the process. This menas that in general the manuals converted last are better HTML than the manuals converted first. That is the containers, basiclib and process manuals may need a little more work than the other manuals. Here is a suggestion of things to check in the manuals:
    1. Check that BETA code fragments are presented in <PRE CLASS=beta> ... </PRE> with the following substitutions made in the body:
          & -> &amp;
          < -> &lt;
          > -> &gt;
       as described in doc/templates/betapresentation.html. This was not automatically done for the first manuals converted. If you have a number of existing <PRE> markups (i.e. without the CLASS=beta), you may use miadoc-replace9 in doc/admin/miadoc-replace.el. This is an Etags based Emacs function, which implements appropriate replacements acros all the involved files.
    2. Check that demo programs are actually present in the code. At least for the basiclib manual, some demos where included in Word by reference, and some of these did not survive through the whole process Word->FrameMaker->WebMaker.
    3. Check that enumerations are marked up with <UL><LI>...</UL>. The Word style Itemize has been automatically converted into this, but not for the first documents converted. There each line became a separate paragraph in HTML. Looks bad.
    4. Maybe some of the first converted manuals has gives rise to too many HTML files. In the last converted manuals, only Heading 1 and Heading 2 yielded new files, but in the first converted manuals, all heading levels yielded new files. Merge HTML files by moving the contents to another file, then fixing all references to the now emptied file (use, e.g. tags-query-replace in Emacs.
    5. Check images. You may want to add ALIGN={left,middle,right} to the images. Also you may want to make background transparent. Most often this can be done by the UNIX command giftrans -T foo.gif > xxx.gif; mv xxx.gif foo.gif. Also some of the images converted by WebMaker sometimes look a little "blurred". Fix this by copying the original images from the original Word files (still present in /users/beta/doc/miadoc/REPOSITORY) into a graphics program, that can save as GIF (e.g. GraphicConverted on Macintosh). You may also want to add WIDTH and HEIGHT attributes to image tags to fasten up the browser's layout process.
    6. Fix references/bibliography. As specified in doc/admin/rules.html references to "our own stuff" should be via the shared doc/bibliography/index.html file. Most of the original Word files contained a list of references to [Madsen 93], [Knudsen 94], [MIA XX-XX]. These should be eliminated from the "private" Bibliographies (and if the Bibliography is then empty, ithe file should be completely eliminated). Use, e.g., the functions miadoc-replace8, miadoc-replace13, and miadoc-replace14 in doc/admin/miadoc-replace.el. These are Etags based Emacs functions, which implements appropriate replacements acros all the involved files.
    7. The WebMaker generated files often contain a "local TOC" for subsections (either in same file as section, or in separate files). I think these may confuse people, and should probably be removed? They are marked with <!-- TOC --> in the HTML.
