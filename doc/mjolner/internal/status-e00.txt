Evaluator-ting:

A. Synlighed:

1. Preference "Open Evaluators in Object Views", der er underpreference til
   "Dynamic compilation" (dvs kan kun enables hvis dynamic eval er enabled.
   Begge disse preferencer skal per default være enabled, dvs alle objekt
   views åbner pr default med evaluator tilknyttet

B. Brugbarhed:

1. Så vidt jeg kan se er der en fast størrelse for
   fragment vinduet i browseren. Jeg kan ihvertfald ikke finde
   ud af at lave resize. Kunne dette vindue ikke være adaptivt
   og tilpasse sig det aktuelle indhold?

2. I Valhalla objekter ser det ud til at 'fit to window'
   kun fitter bredden og ikke højden.

3. Og bredden ser du til at blive beregnet forkert for origin
   idet de sidste 1-2 tegn ikke kan ses

4. Og teksten i ORIGIN-feltet er meget verbose.

5. Default evaluator er "do <<ImpOpt>>" i stedet for <<DopartOpt>>
6. Tryk på Evaluate knappen checker om editor er i text edit mode, og
   forlader automatisk dette, hvis tilfældet
7. Shortcut for Evaluate knappen. Kunne vel være samme shortcut som
   "Run" har når man ikke debugger?
8. ObjectViews skal opdatere sig efter hver evaluator

C. Debug
1. Det ville lette debug af valhalla meget hvis de gamle options
   på kommandolinien kunne bringes i sving igen. Dvs få --valhalla til
   at virke. Specielt --valhalla -env BETART.
